# Smart Teacher (智导) - 产品需求文档 (PRD) v2.0

## 1. 项目概述
**项目名称**：Smart Teacher (智导)
**核心理念**：为用户打造专属的AI导师团队，通过“建立全维档案-匹配双层导师-全程闭环指导”的模式，辅助用户高效实现生活或工作中的项目目标。
**目标用户**：有自我提升需求、创业想法、或希望在特定领域获得指导的个人用户。
**平台性质**：**微信小程序** (优先)，后续适配H5及Web端。

---

## 2. 用户流程与功能模块

### 第一步：建立全维数字档案 (Holistic Digital Profile)
**目标**：构建用户的“数字孪生”，确保AI不仅仅了解性格，更了解能力与资源。
1.  **多维数据采集**：
    *   **心理与人格 (Mindset)**：通过MBTI、大五人格等变体测试，了解内在驱动力与性格优劣势。
    *   **做事风格 (Execution Style)**：
        *   **决策偏好**：测试用户是“直觉型”还是“数据型”，“冒险型”还是“稳健型”。
        *   **行动力评估**：通过简单的场景模拟题（如“遇到困难你会先找人还是先查资料？”），判断执行力模式。
    *   **资源盘点 (Resources)**：
        *   **人脉资源**：引导用户标签化输入核心人脉（如“有媒体朋友”、“有供应链渠道”）。
        *   **技能树**：用户勾选或输入已掌握的技能（如“擅长写作”、“懂财务”）。
        *   **资金/时间预算**：明确项目可投入的资金范围和每周可用时间。
    *   **认知边界 (Cognition)**：通过对几个热点事件的看法，评估用户的商业认知水平。
2.  **采集方式**：
    *   **AI交互式访谈**：像聊天一样自然获取信息。
    *   **多媒体解析**：支持上传个人朋友圈截图、抖音/视频号链接，AI分析其生活状态与兴趣偏好。

### 第二步：建立导师数据库——导师智能体 (Mentor Agents)
**目标**：构建“高大上”与“接地气”并存的导师矩阵。
1.  **双层导师体系**：
    *   **宏观导师 (战略层)**：行业领袖（如马斯克、张一鸣），提供顶层思维、愿景与战略方向。
    *   **实战导师 (战术层)**：本土化成功者（如隔壁旺铺老板、某垂直领域KOL），提供落地细节、避坑指南与实操技巧。
2.  **导师构建与匹配**：
    *   **名人自动构建**：输入姓名，AI自动联网检索演讲、书籍、访谈，提炼思维模型。
    *   **身边人手动构建**：用户输入非名人资料（语录、经验），AI结构化存储。
    *   **智能推荐**：基于用户的“全维档案”和“项目目标”，推荐互补型导师（例如：执行力差的用户匹配强督促型导师）。

### 第三步：核心链接——全程智导 (Smart Guidance Loop)
**目标**：在执行前、中、后全程介入，缩小用户与导师的差距。
1.  **执行前：精准规划**
    *   AI结合用户“资源盘点”和导师“成功模型”，生成**可行性极高**的To-Do List。
    *   *差异化策略*：资金少的用户，实战导师推荐低成本启动方案；资金足的用户，宏观导师推荐高举高打方案。
2.  **执行中：动态修正**
    *   **实时指导**：用户记录过程，AI对比导师标准。
    *   **圆桌会议**：AI模拟导师间争论（如宏观导师说“要扩张”，实战导师说“先稳住现金流”），最终给出综合建议。
    *   **智能评分系统 (0-10分)**：
        *   **9-10分**：收录进个人成功库。
        *   **4-7分**：AI指出具体偏差，提供修正步骤。
        *   **0-3分**：启动“手把手教学”，拆解为婴儿步（Baby Steps）。
    *   **2/8法则兜底**：差距过大时，只锁定20%关键动作。
3.  **督导机制**：
    *   基于用户“做事风格”设定提醒频率（拖延症用户高频提醒）。

## 3. 核心辅助功能模块

### 3.1 实时反馈与导师评估机制
**目标**：建立项目执行过程中的动态反馈闭环。
1.  **实时进度上报**：
    *   用户随时可以向系统汇报项目进展（文字、图片、语音）。
    *   系统自动记录时间线，形成项目日志。
2.  **智能评估系统**：
    *   AI调用所有导师智能体，根据用户上报内容进行**多维度评分**（执行速度、方向准确度、资源利用率）。
    *   自动生成**差距分析报告**：对比导师在同阶段的做法，指出关键差异。
    *   **逾期预警**：任务未完成时，每日提醒并展示与标准进度的差距。
    *   **督促语气定制**：根据用户"做事风格"（拖延型 vs 自律型）调整提醒语气（温和 vs 严厉）。
3.  **智能调整**：
    *   若用户多次延期，AI自动调整计划（启用2/8法则简化任务）。

### 3.3 社交分享与协作机制
**目标**：支持多人协作项目，增强传播裂变。
1.  **项目协作分享**：
    *   **生成分享链接**：项目发起人可生成专属链接，邀请协作者加入。
    *   **角色分配**：支持设置"项目负责人"、"执行成员"、"观察者"等角色，权限差异化。
    *   **协作看板**：所有成员共享项目进度、任务分配、导师建议。
2.  **朋友圈/社交分享**：
    *   **成果卡片生成**：用户完成阶段性目标后，系统自动生成精美海报（含项目名、完成度、导师评分），一键分享朋友圈。
    *   **小程序分享**：支持分享小程序首页/个人项目页给微信好友。
3.  **推荐奖励机制**：
    *   **邀请新用户**：每成功邀请1位新用户注册，奖励邀请人**额外3次免费对话**。
    *   **付费转化奖励**：邀请的用户若购买会员，邀请人获得**1个月免费会员**延期。
    *   **排行榜**：展示本周/本月邀请人数最多的前10名用户，激发竞争氛围。

### 3.4 管理员系统 (Admin Dashboard)
**目标**：为超级管理员提供全局管控能力。
1.  **超级管理员账号**：
    *   **唯一账号**：`wanghao304`  
    *   **唯一密码**：`lm888456`  
    *   **安全措施**：强制开启双因素认证（绑定手机/邮箱）。
2.  **核心管理功能**：
    *   **用户管理**：
        *   查看所有用户列表（含注册时间、会员状态、活跃度）。
        *   手动调整用户权益（如赠送会员、增加免费次数）。
        *   封禁/解封用户账号。
    *   **内容审核**：
        *   审查用户上传的导师资料（防止违规内容）。
        *   审核用户分享的项目海报。
    *   **数据分析**：
        *   实时查看平台关键指标（DAU、MAU、付费转化率、Token消耗）。
        *   导师智能体使用频率统计（哪些导师最受欢迎）。
        *   用户项目完成率、平均评分等。
    *   **财务管理**：
        *   查看收入报表（会员购买、充值记录）。
        *   Token成本监控（实时/月度）。
    *   **系统配置**：
        *   调整会员定价、免费额度。
        *   配置推送通知模板。
        *   管理豆包API密钥。
3.  **管理界面要求**：
    *   **独立Web管理后台**（不在小程序内）。
    *   支持图表可视化（ECharts）。
    *   响应式设计，适配电脑/平板。

## 4. 盈利模式与会员体系 (Smart Membership)

### 4.1 成本模型 (基于豆包大模型)
*   **Token消耗预估**：单次深度交互约1.5k-2k Tokens。
*   **免费用户策略**：
    *   **额度**：每日3次免费对话/指导。
    *   **逻辑**：利用免费用户产生的数据优化模型，同时作为付费转化的基数。
*   **付费会员 (Pro)**：
    *   **定价**：9.9元/年 (早鸟价)。
    *   **权益**：每日30次对话，解锁无限导师位，生成深度分析报告。
*   **收支平衡点**：
    *   依靠“低活跃度免费用户”分摊成本。
    *   1个付费用户（9.9元）约可覆盖15个普通活跃度的免费用户成本。

## 4. 技术架构 (全权委托)

### 4.1 核心技术栈
*   **前端框架**：**Taro** 或 **Uni-app** (React语法)，实现**微信小程序**原生体验，同时支持编译为H5/App。
*   **后端服务**：**Supabase** (Serverless)
    *   PostgreSQL 数据库：存储用户档案、导师数据。
    *   Edge Functions：处理复杂的业务逻辑（如评分、推荐）。
*   **AI 引擎**：
    *   **模型**：豆包 (Doubao-seed-1-6-251015)。
    *   **Agent框架**：LangChain.js (用于构建导师智能体、RAG检索)。
*   **部署**：Vercel (后端/H5) + 微信小程序后台。

### 4.2 数据流向
1.  用户在小程序输入 -> 调用Supabase Edge Function。
2.  Edge Function 组装Prompt (包含用户档案 + 导师模型) -> 调用豆包API。
3.  豆包返回结果 -> Edge Function 解析并存库 -> 小程序前端渲染。
